# 피처 사전 (Feature Dictionary)

이커머스 이탈 예측 시스템에서 사용하는 30개 이상의 피처를 정의한 문서입니다.
피처는 `src/features/feature_engineering.py`의 `FeatureEngineer` 클래스에서 계산됩니다.

---

## 1. RFM 피처 (Recency, Frequency, Monetary)

RFM은 고객 가치를 정량화하는 마케팅 분석의 기본 프레임워크입니다.
구매 이벤트(`event_type == "purchase"`)를 기반으로 계산됩니다.

### 1.1 recency

| 항목 | 내용 |
|------|------|
| **타입** | float |
| **계산 방법** | `(기준일 - 마지막 구매일).days`. 구매 이력 없는 고객은 365로 설정. `clip(lower=0)` 적용 |
| **범위** | 0 ~ 365 (일) |
| **비즈니스 의미** | 마지막 구매 이후 경과 일수. 값이 클수록 최근에 구매하지 않은 이탈 위험 고객 |
| **설계 근거** | 고객이 마지막으로 구매한 시점은 이탈 가능성의 가장 직접적인 신호. 최대 365일로 제한하여 극단치 제거 |

### 1.2 frequency

| 항목 | 내용 |
|------|------|
| **타입** | int |
| **계산 방법** | 전체 기간 구매 이벤트 발생 횟수의 합 |
| **범위** | 0 이상 |
| **비즈니스 의미** | 총 구매 횟수. 충성도 높은 고객일수록 높은 값 |
| **설계 근거** | 구매 빈도는 고객과 서비스 사이의 관계 깊이를 나타내는 핵심 지표 |

### 1.3 monetary

| 항목 | 내용 |
|------|------|
| **타입** | float |
| **계산 방법** | 전체 구매 이벤트의 `amount` 합계 |
| **범위** | 0.0 이상 (원화) |
| **비즈니스 의미** | 누적 구매 금액. 고가치 고객 식별에 사용 |
| **설계 근거** | 매출 기여도가 높은 고객에게 더 높은 리텐션 비용을 투자하는 것이 합리적 |

### 1.4 avg_order_value

| 항목 | 내용 |
|------|------|
| **타입** | float |
| **계산 방법** | `monetary / frequency`. frequency == 0이면 0.0 |
| **범위** | 0.0 이상 (원화) |
| **비즈니스 의미** | 평균 주문 금액. 단건 구매 성향과 구매력을 나타냄 |
| **설계 근거** | 총 구매액과 빈도를 함께 고려해야 실질적인 객단가 파악 가능 |

### 1.5 monetary_per_day

| 항목 | 내용 |
|------|------|
| **타입** | float |
| **계산 방법** | `monetary / (365 - recency + 1)`. 활성 기간 대비 일평균 매출 |
| **범위** | 0.0 이상 |
| **비즈니스 의미** | 활성 기간 동안의 일평균 매출 기여도 |
| **설계 근거** | 구매 기간을 보정하여 장기 저빈도 고객과 단기 고빈도 고객을 구분 |

## 2. 행동 변화율 피처 (Behavioral Change Features)

고객의 활동 기간을 중앙값 기준으로 전반부(first half)와 후반부(second half)로 분할하여,
각 지표의 변화율을 측정합니다. 계산 공식: `second_rate / first_rate` (0으로 나누는 경우 0.0 반환).

### 2.1 visit_frequency_change

| 항목 | 내용 |
|------|------|
| **타입** | float |
| **계산 방법** | `(후반 일별 page_view 수) / (전반 일별 page_view 수)` |
| **범위** | 0.0 이상 (1.0 = 변화 없음, < 1.0 = 감소) |
| **비즈니스 의미** | 방문 빈도 변화율. 1.0 미만은 이탈 경향 시사 |
| **설계 근거** | 방문 감소는 이탈의 선행 지표로 가장 명확한 신호 중 하나 |

### 2.2 purchase_cycle_change

| 항목 | 내용 |
|------|------|
| **타입** | float |
| **계산 방법** | `(후반 일별 구매 수) / (전반 일별 구매 수)` |
| **범위** | 0.0 이상 |
| **비즈니스 의미** | 구매 주기 변화율. 감소는 구매 의욕 저하를 나타냄 |
| **설계 근거** | 구매 빈도 변화는 recency보다 빠른 이탈 선행 지표 역할 |

### 2.3 session_duration_change

| 항목 | 내용 |
|------|------|
| **타입** | float |
| **계산 방법** | `session_duration` 컬럼 존재 시 평균값 비율; 없으면 일별 이벤트 수 비율 |
| **범위** | 0.0 이상 |
| **비즈니스 의미** | 세션 체류시간 변화율. 감소는 콘텐츠 흥미 저하 의미 |
| **설계 근거** | 체류시간 감소는 구매 감소보다 선행하여 나타나는 경우가 많음 |

### 2.4 cart_conversion_change

| 항목 | 내용 |
|------|------|
| **타입** | float |
| **계산 방법** | `(후반 장바구니→구매 전환율) / (전반 장바구니→구매 전환율)` |
| **범위** | 0.0 이상 |
| **비즈니스 의미** | 장바구니 전환율 변화. 감소는 구매 결정 장벽이 높아졌음을 시사 |
| **설계 근거** | 가격 민감도 상승이나 만족도 하락이 전환율로 발현됨 |

### 2.5 coupon_response_change

| 항목 | 내용 |
|------|------|
| **타입** | float |
| **계산 방법** | `(후반 일별 coupon_use 수) / (전반 일별 coupon_use 수)` |
| **범위** | 0.0 이상 |
| **비즈니스 의미** | 쿠폰 사용 변화율. 증가는 가격 민감도 상승, 감소는 이탈 임박 신호 |
| **설계 근거** | 쿠폰 반응도 변화로 프로모션 의존도와 이탈 위험을 동시에 포착 |

### 2.6 search_intensity_change

| 항목 | 내용 |
|------|------|
| **타입** | float |
| **계산 방법** | `(후반 일별 search 수) / (전반 일별 search 수)` |
| **범위** | 0.0 이상 |
| **비즈니스 의미** | 검색 강도 변화율. 감소는 탐색 의도 저하 |
| **설계 근거** | 검색 활동은 구매 의향의 선행 신호이므로 감소 추적이 중요 |

### 2.7 cs_contact_change

| 항목 | 내용 |
|------|------|
| **타입** | float |
| **계산 방법** | `(후반 일별 cs_contact 수) / (전반 일별 cs_contact 수)` |
| **범위** | 0.0 이상 |
| **비즈니스 의미** | 고객 센터 문의 변화율. 급격한 증가는 불만족 신호 |
| **설계 근거** | 불만 고객은 이탈 직전에 CS 문의가 증가하는 경향이 있음 |

## 3. 구매 주기 이상 피처 (Purchase Cycle Anomaly)

### 3.1 purchase_cycle_anomaly

| 항목 | 내용 |
|------|------|
| **타입** | float |
| **계산 방법** | `days_since_last_purchase / avg_purchase_cycle_days`. 구매 없으면 2.0, 구매 1회면 1.5 |
| **범위** | 0.0 이상 (1.0 = 평균 주기 도달, > 1.0 = 구매가 늦어짐) |
| **비즈니스 의미** | 평균 구매 주기 대비 현재 경과 일수의 비율. 이탈 위험 조기 경보 |
| **설계 근거** | 절대적인 날짜보다 고객별 상대적 주기로 측정하면 개인화된 위험 탐지 가능 |

### 3.2 avg_purchase_cycle_days

| 항목 | 내용 |
|------|------|
| **타입** | float |
| **계산 방법** | 구매 간 날짜 차이(`np.diff`)의 평균 |
| **범위** | 0.0 이상 (일) |
| **비즈니스 의미** | 고객의 통상적인 구매 간격 |
| **설계 근거** | 개인별 기준선을 구축하여 이상 탐지의 정밀도 향상 |

### 3.3 days_since_last_purchase

| 항목 | 내용 |
|------|------|
| **타입** | float |
| **계산 방법** | `(기준일 - 마지막 구매일).days` |
| **범위** | 0.0 이상 (일) |
| **비즈니스 의미** | 마지막 구매 이후 경과 일수 (이상 탐지의 분자) |
| **설계 근거** | recency와 유사하나 purchase_cycle_anomaly 계산의 원시 값으로 별도 보존 |

## 4. 세션 품질 피처 (Session Quality Features)

### 4.1 avg_session_duration

| 항목 | 내용 |
|------|------|
| **타입** | float |
| **계산 방법** | `session_duration` 컬럼 존재 시 평균; 없으면 고객별 일평균 이벤트 수 |
| **범위** | 0.0 이상 |
| **비즈니스 의미** | 평균 세션 체류 시간. 높을수록 콘텐츠에 대한 관심도가 높음 |
| **설계 근거** | 체류 시간은 구매 의향과 강한 양의 상관관계를 보임 |

### 4.2 pageviews_per_session

| 항목 | 내용 |
|------|------|
| **타입** | float |
| **계산 방법** | `page_view 이벤트 수 / 방문 일수` |
| **범위** | 0.0 이상 |
| **비즈니스 의미** | 세션당 페이지뷰 수. 탐색 깊이를 나타냄 |
| **설계 근거** | 단순 방문 횟수보다 방문의 질을 반영하는 지표 |

### 4.3 search_to_purchase_rate

| 항목 | 내용 |
|------|------|
| **타입** | float |
| **계산 방법** | `구매 수 / 검색 수`. 검색 없으면 0.0 |
| **범위** | 0.0 ~ 1.0 이상 |
| **비즈니스 의미** | 검색 대비 구매 전환율. 탐색→구매 효율성을 측정 |
| **설계 근거** | 낮은 전환율은 탐색 목적이 구매가 아니거나 상품 매칭 문제를 시사 |

### 4.4 cart_abandonment_rate

| 항목 | 내용 |
|------|------|
| **타입** | float |
| **계산 방법** | `remove_from_cart 수 / add_to_cart 수`. 장바구니 추가 없으면 0.0 |
| **범위** | 0.0 ~ 1.0 이상 |
| **비즈니스 의미** | 장바구니 이탈율. 높을수록 구매 결정 장애 요인 존재 |
| **설계 근거** | 장바구니 이탈은 가격, UX, 신뢰도 문제의 복합적 신호 |

### 4.5 avg_events_per_day

| 항목 | 내용 |
|------|------|
| **타입** | float |
| **계산 방법** | `전체 이벤트 수 / 활동 기간(일)` |
| **범위** | 0.0 이상 |
| **비즈니스 의미** | 일평균 활동량. 플랫폼 참여 강도를 종합 측정 |
| **설계 근거** | 특정 이벤트 유형에 치우치지 않고 전반적인 활성화 수준 파악 |

## 5. 시퀀스 피처 (Sequence Features)

### 5.1 sequence_diversity

| 항목 | 내용 |
|------|------|
| **타입** | int |
| **계산 방법** | 고객이 발생시킨 고유 이벤트 유형 수 (`nunique`) |
| **범위** | 0 ~ 8 (page_view, search, add_to_cart, remove_from_cart, purchase, coupon_use, review, cs_contact) |
| **비즈니스 의미** | 행동 다양성. 높을수록 플랫폼과 다면적으로 상호작용하는 충성 고객 |
| **설계 근거** | 다양한 기능을 사용하는 고객은 이탈 가능성이 낮다는 연구 결과를 반영 |

### 5.2 sequence_length

| 항목 | 내용 |
|------|------|
| **타입** | int |
| **계산 방법** | 전체 이벤트 수 |
| **범위** | 0 이상 |
| **비즈니스 의미** | 총 상호작용 횟수. 참여 규모를 나타냄 |
| **설계 근거** | 기간이 같더라도 이벤트 수가 많은 고객이 더 활성화된 상태 |

### 5.3 behavior_pattern_cluster

| 항목 | 내용 |
|------|------|
| **타입** | int |
| **계산 방법** | 이벤트 유형 분포 벡터(8차원)에 K-Means(k=5) 클러스터링 적용 결과 |
| **범위** | 0 ~ 4 |
| **비즈니스 의미** | 행동 패턴 군집 ID. 유사한 사용 패턴을 가진 고객 그룹 식별 |
| **설계 근거** | 이벤트 비율 벡터로 클러스터링하면 개인 차이를 넘는 집단적 패턴 포착 가능 |

### 5.4 purchase_sequence_trend

| 항목 | 내용 |
|------|------|
| **타입** | float |
| **계산 방법** | 구매 순서와 금액 간의 Pearson 상관계수 (`np.corrcoef`). 구매 2회 미만이면 0.0 |
| **범위** | -1.0 ~ 1.0 |
| **비즈니스 의미** | 구매 금액의 시간적 추세. 양수 = 증가 추세, 음수 = 감소 추세 |
| **설계 근거** | 구매 금액 감소 추세는 이탈 전 단계에서 자주 관찰되는 패턴 |

## 6. 시간대 피처 (Time-Based Features)

### 6.1 weekend_purchase_ratio

| 항목 | 내용 |
|------|------|
| **타입** | float |
| **계산 방법** | `주말 구매 수 / 전체 구매 수` (dayofweek >= 5가 주말) |
| **범위** | 0.0 ~ 1.0 |
| **비즈니스 의미** | 주말 구매 비율. 직장인 vs 자영업자 등 라이프스타일 구분에 활용 |
| **설계 근거** | 구매 시점 패턴은 고객 생활 리듬을 반영하여 맞춤 마케팅 타이밍 최적화에 유용 |

### 6.2 morning_activity_ratio

| 항목 | 내용 |
|------|------|
| **타입** | float |
| **계산 방법** | `오전(6~12시) 이벤트 수 / 전체 이벤트 수` (timestamp 컬럼 필요) |
| **범위** | 0.0 ~ 1.0 |
| **비즈니스 의미** | 오전 활동 비율 |
| **설계 근거** | 시간대별 활동 패턴으로 개인화된 푸시 알림 최적 발송 시간 결정 |

### 6.3 evening_activity_ratio

| 항목 | 내용 |
|------|------|
| **타입** | float |
| **계산 방법** | `저녁(18~22시) 이벤트 수 / 전체 이벤트 수` |
| **범위** | 0.0 ~ 1.0 |
| **비즈니스 의미** | 저녁 활동 비율. 한국 이커머스 이용자의 피크 타임 |
| **설계 근거** | 저녁 집중 이용자는 퇴근 후 쇼핑 습관을 가진 직장인일 가능성이 높음 |

### 6.4 night_activity_ratio

| 항목 | 내용 |
|------|------|
| **타입** | float |
| **계산 방법** | `야간(22시 이후 또는 6시 이전) 이벤트 수 / 전체 이벤트 수` |
| **범위** | 0.0 ~ 1.0 |
| **비즈니스 의미** | 심야 활동 비율 |
| **설계 근거** | 심야 이용자 세그먼트는 별도 마케팅 전략이 필요한 고객군 |

### 6.5 peak_hour

| 항목 | 내용 |
|------|------|
| **타입** | int |
| **계산 방법** | 이벤트 발생 시각(hour)의 최빈값 (`mode`) |
| **범위** | 0 ~ 23 |
| **비즈니스 의미** | 가장 활동이 많은 시간대 |
| **설계 근거** | 개인별 피크 시간을 파악하여 최적 마케팅 발송 시점 결정 |

### 6.6 day_variance

| 항목 | 내용 |
|------|------|
| **타입** | float |
| **계산 방법** | 날짜별 이벤트 수의 분산 (`var`) |
| **범위** | 0.0 이상 |
| **비즈니스 의미** | 일별 활동량의 편차. 높을수록 불규칙한 이용 패턴 |
| **설계 근거** | 활동의 일관성은 플랫폼 습관화 수준을 나타냄 |

## 7. 고객 여정 단계 피처 (Journey Stage Features)

### 7.1 journey_stage

| 항목 | 내용 |
|------|------|
| **타입** | int (카테고리형) |
| **계산 방법** | 최근 30일 vs 이전 기간의 이벤트 비율과 구매 수를 기반으로 규칙 분류 |
| **범위** | 0=onboarding, 1=growing, 2=mature, 3=declining, 4=dormant |
| **비즈니스 의미** | 고객의 현재 라이프사이클 단계 |
| **설계 근거** | 단계별로 최적의 개입 전략이 다름. 예: growing 단계는 장려, declining은 재유입 |

분류 기준:
- **onboarding (0)**: 활동 기간 30일 미만
- **growing (1)**: 최근 활동률이 이전 대비 1.2배 이상 증가
- **mature (2)**: 안정적 활동(±20% 이내)이며 구매 3회 이상
- **declining (3)**: 최근 활동률이 이전 대비 50% 미만
- **dormant (4)**: 최근 일평균 활동 0.1 미만

### 7.2 stage_tenure_days

| 항목 | 내용 |
|------|------|
| **타입** | int |
| **계산 방법** | 현재 여정 단계에서 머문 추정 일수 |
| **범위** | 0 이상 (일) |
| **비즈니스 의미** | 현재 단계 체류 기간. 오래될수록 전환 확률 변화 |
| **설계 근거** | declining 단계 체류 일수가 길수록 이탈 위험도 증가 |
