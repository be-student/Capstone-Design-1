# A/B 테스트 리포트 (A/B Test Report)

리텐션 캠페인의 효과를 과학적으로 검증하는 A/B 테스트 방법론과 결과 해석 가이드입니다.
구현 코드: `src/analysis/ab_testing.py`, `src/models/ab_testing.py`

---

## 1. A/B 테스트 프레임워크 개요

### 1.1 구성 요소

| 구성 요소 | 클래스/함수 | 역할 |
|----------|-----------|------|
| 표본 크기 산출 | `PowerAnalysis.required_sample_size()` | Power Analysis 기반 필요 샘플 계산 |
| 그룹 배정 | `ABTestFramework.assign_groups()` | 처우/통제 그룹 무작위 배정 |
| 통계 검정 | `ABTestFramework.compute_significance()` | Z-test, Chi-square, t-test 자동 선택 |
| 다중 비교 보정 | `MultipleComparisonCorrection` | Bonferroni, FDR(BH), Holm-Bonferroni |

### 1.2 실험 설계 원칙

- **처우 비율**: 50% (treatment_ratio = 0.50, 동등 배분)
- **유의 수준**: α = 0.05
- **검정력**: Power ≥ 0.80 (최소 80% 이상 요구)
- **양측 검정**: 기본값 (two_sided=True)
- **그룹 배정**: 결정론적 해싱 기반 (동일 seed로 재현 가능)

## 2. Power Analysis (검정력 분석)

### 2.1 필요 표본 크기 산출

**목적:** 지정한 효과 크기를 검출하기에 충분한 표본 크기를 사전에 결정합니다.

**산출 공식 (`PowerAnalysis.required_sample_size`):**

$$n = \left\lceil \frac{\left(z_{\alpha/2}\sqrt{2\bar{p}(1-\bar{p})} + z_{\beta}\sqrt{p_1(1-p_1)+p_2(1-p_2)}\right)^2}{(p_2-p_1)^2} \right\rceil$$

여기서:
- $p_1$ = 기준 이탈율 (baseline_rate)
- $p_2$ = $p_1$ + MDE (처우 후 기대 이탈율)
- $\bar{p}$ = $(p_1 + p_2) / 2$ (통합 이탈율)
- $z_{\alpha/2}$ = 양측 검정 임계값 (`norm.ppf(1 - α/2)`, α=0.05이면 ≈ 1.96)
- $z_{\beta}$ = 검정력 임계값 (`norm.ppf(power)`, power=0.80이면 ≈ 0.84)

### 2.2 파라미터 정의

| 파라미터 | 기본값 | 의미 |
|---------|-------|------|
| `baseline_rate` | - | 현재 이탈율 (필수 입력) |
| `mde` | - | 최소 탐지 효과 크기 (Minimum Detectable Effect, 절대 차이) |
| `alpha` | 0.05 | 1종 오류율 (유의 수준) |
| `power` | 0.80 | 검정력 (1 - 2종 오류율 β) |
| `two_sided` | True | 양측 검정 여부 |

### 2.3 산출 예시

| 기준 이탈율 | MDE | α | Power | 그룹당 필요 표본 | 총 표본 |
|-----------|-----|---|-------|--------------|--------|
| 0.30 (30%) | 0.05 (5%p) | 0.05 | 0.80 | ≈ 1,026명 | ≈ 2,052명 |
| 0.30 (30%) | 0.03 (3%p) | 0.05 | 0.80 | ≈ 2,878명 | ≈ 5,756명 |
| 0.30 (30%) | 0.05 (5%p) | 0.05 | 0.90 | ≈ 1,374명 | ≈ 2,748명 |
| 0.20 (20%) | 0.05 (5%p) | 0.05 | 0.80 | ≈ 826명 | ≈ 1,652명 |

**해석:** 현재 이탈율이 30%일 때 5%p 개선을 검출하려면 그룹당 최소 1,026명이 필요합니다.
MDE를 줄일수록(더 작은 효과도 탐지하려면) 필요 표본이 급격히 증가합니다.

### 2.4 역방향 산출: 최소 탐지 효과 (MDE)

기존 표본 크기에서 탐지 가능한 최소 효과 크기를 역산합니다:
`PowerAnalysis.minimum_detectable_effect(n, baseline_rate, alpha, power)`

이진 탐색(100회 반복)으로 목표 검정력을 달성하는 최소 효과 크기 도출.

## 3. 그룹 배정 방법

### 3.1 무작위 배정 알고리즘

```python
# 재현 가능한 무작위 배정
rng = np.random.RandomState(seed)
indices = rng.permutation(n)
n_treatment = int(round(n * treatment_ratio))  # 50%
# 처음 n_treatment 인덱스를 처우 그룹으로 배정
```

**핵심 특성:**
- 동일 seed로 항상 동일한 배정 결과 (결정론적)
- 처우/통제 비율을 정확하게 유지
- A/B/n 다변량 실험도 지원 (`n_variants > 2`)

### 3.2 실험군 구성

| 그룹 | 정의 | 처우 내용 |
|------|------|---------|
| **처우 (Treatment)** | 전체의 50% | 리텐션 캠페인 수신 (쿠폰/VIP Care/푸시) |
| **통제 (Control)** | 전체의 50% | 캠페인 없음 (자연 이탈 관찰) |

## 4. 통계 검정 방법

### 4.1 자동 검정 유형 선택

`compute_significance()` 함수는 지표의 데이터 유형에 따라 적절한 검정을 자동 선택합니다:

```
지표 값이 {0, 1}만 포함 → 이진형 → Z-test (비율 검정)
그 외 연속형 지표       → 연속형 → Welch's t-test
```

### 4.2 Z-test (비율 차이 검정)

**적용 상황:** 이탈 여부(0/1), 구매 여부(0/1) 등 이진 결과 지표

**귀무가설:** $H_0: p_{\text{treatment}} = p_{\text{control}}$

**검정 통계량:**

$$z = \frac{\hat{p}_T - \hat{p}_C}{\sqrt{\hat{p}(1-\hat{p})\left(\frac{1}{n_T} + \frac{1}{n_C}\right)}}$$

여기서:
- $\hat{p}_T$, $\hat{p}_C$: 각 그룹의 이탈율
- $\hat{p}$: 통합 이탈율 $(n_T \hat{p}_T + n_C \hat{p}_C) / (n_T + n_C)$
- $n_T$, $n_C$: 처우/통제 그룹 표본 수

### 4.3 Chi-square 검정 (카이제곱 독립성 검정)

**적용 상황:** 이진 지표의 보조 검정 (빈도표 기반)

**2×2 분할표:**

| | 이탈 | 비이탈 |
|-|------|-------|
| **처우 그룹** | $o_{11}$ | $o_{12}$ |
| **통제 그룹** | $o_{21}$ | $o_{22}$ |

$$\chi^2 = \sum \frac{(O_{ij} - E_{ij})^2}{E_{ij}}$$

자유도 = 1, 유의 수준 α = 0.05

### 4.4 Welch's t-test (독립 표본 t-test)

**적용 상황:** 구매 금액, 세션 시간, CLV 등 연속형 지표

**귀무가설:** $H_0: \mu_{\text{treatment}} = \mu_{\text{control}}$

**검정 통계량:**

$$t = \frac{\bar{X}_T - \bar{X}_C}{\sqrt{\frac{s_T^2}{n_T} + \frac{s_C^2}{n_C}}}$$

Welch-Satterthwaite 자유도로 등분산 가정 불필요 (안정적).

### 4.5 Mann-Whitney U 검정 (비모수)

**적용 상황:** 정규성 가정이 불확실한 연속형 지표

`StatisticalTestSuite.mann_whitney_u_test()`로 제공.

## 5. 95% 신뢰구간 및 p-value 산출

### 5.1 p-value

$$p\text{-value} = P(|Z| \geq |z_{\text{obs}}|\ |\ H_0)$$

양측 검정: $p = 2 \times (1 - \Phi(|z|))$

- `p < 0.05`: 통계적으로 유의미 (귀무가설 기각)
- `p ≥ 0.05`: 통계적으로 유의미하지 않음

### 5.2 95% 신뢰구간 (비율 차이)

$$CI_{95\%} = (\hat{p}_T - \hat{p}_C) \pm 1.96 \times \sqrt{\frac{\hat{p}_T(1-\hat{p}_T)}{n_T} + \frac{\hat{p}_C(1-\hat{p}_C)}{n_C}}$$

**해석:**
- CI가 0을 포함하지 않으면 → 효과 유의미
- CI의 하한이 양수이면 → 처우 그룹이 통계적으로 우월

### 5.3 효과 크기 (Effect Size)

| 지표 유형 | 효과 크기 측정 | 공식 |
|---------|-------------|------|
| 이진형 | Cohen's h | $h = 2\arcsin(\sqrt{p_T}) - 2\arcsin(\sqrt{p_C})$ |
| 연속형 | Cohen's d | $d = (\bar{X}_T - \bar{X}_C) / s_{pooled}$ |

Cohen's d 해석 기준: 소(0.2), 중(0.5), 대(0.8)

## 6. 다중 비교 보정

### 6.1 필요성

동일 실험에서 여러 지표를 동시에 검정하면 1종 오류(위양성) 누적:
- k개 검정 시 적어도 하나 위양성 확률: $1 - (1-\alpha)^k$
- k=10, α=0.05: $1 - 0.95^{10} \approx 40\%$ (실제 유의수준이 40%로 팽창)

### 6.2 보정 방법

**Bonferroni 보정 (보수적):**
$$\alpha^* = \alpha / k$$
$$p^*_{adj} = \min(p_i \times k, 1.0)$$
- 가족 단위 오류율(FWER) 제어
- 독립적인 검정에 적합

**Benjamini-Hochberg FDR 보정 (균형적):**
$$\text{Rank } i \text{의 임계값}: \alpha \times i / k$$
- 위발견율(FDR) 제어
- 상관된 다수의 검정에 적합
- Bonferroni보다 검정력 높음

**Holm-Bonferroni 보정 (중간):**
- 순서대로 기각하는 단계적 절차
- FWER 제어하면서 Bonferroni보다 강력

## 7. A/B 테스트 결과 해석 예시

### 7.1 쿠폰 캠페인 A/B 테스트 시나리오

**실험 설계:**
- 기준 이탈율: 30%
- MDE: 5%p (처우 후 목표 이탈율 25%)
- α = 0.05, Power = 0.80
- 필요 표본: 그룹당 1,026명 → 총 2,052명

**가상 결과:**

| 그룹 | 표본 수 | 이탈 수 | 이탈율 |
|------|--------|--------|-------|
| 처우 (쿠폰 발송) | 1,026 | 256 | 24.9% |
| 통제 (미발송) | 1,026 | 308 | 30.0% |
| **차이** | - | -52 | **-5.1%p** |

**통계 검정 결과:**
```
Z-statistic: -2.83
p-value: 0.0046
95% 신뢰구간: [-8.7%p, -1.6%p]
Cohen's h: 0.12 (소-중간 효과)
```

**해석:**
- p = 0.0046 < 0.05 → 통계적으로 유의미한 효과 존재
- 95% CI [-8.7%p, -1.6%p]는 0을 포함하지 않음 → 처우 효과 확인
- 이탈율이 30.0% → 24.9%로 **5.1%p 감소** (캠페인 효과)

### 7.2 VIP Care 캠페인 A/B 테스트 시나리오

| 그룹 | 표본 수 | 이탈율 | 평균 CLV 보존 |
|------|--------|-------|-------------|
| 처우 (VIP Care) | 50 | 10% | 285,000원 |
| 통제 (미진행) | 50 | 32% | 180,000원 |

**통계 검정 결과:**
```
Z-statistic: -2.61
p-value: 0.0091
95% 신뢰구간: [-37.9%p, -5.5%p]
```

**비용 편익:**
- 처우 비용: 50명 × 70,000원 = 3,500,000원
- 보존 CLV: (285,000 - 180,000) × 50 = 5,250,000원
- 순 이익: 5,250,000 - 3,500,000 = **1,750,000원**

### 7.3 결과 해석 체크리스트

| 검토 항목 | 기준 | 확인 |
|---------|------|------|
| p-value < 0.05 | 통계적 유의성 | |
| 95% CI가 0 포함하지 않음 | 효과 방향 확인 | |
| 효과 크기가 비즈니스 의미 있음 | 실용적 유의성 | |
| 표본 크기 ≥ Power Analysis 요구량 | 충분한 검정력 | |
| 다중 비교 보정 적용 | 위양성 방지 | |
| 실험 기간 충분 (최소 2주) | 노벨티 효과 배제 | |

## 8. 실험 실행 워크플로우

```
1. Power Analysis
   → required_sample_size(baseline_rate=0.30, mde=0.05)
   → 필요 표본: 1,026명/그룹

2. 그룹 배정
   → assign_groups(customer_ids, n_variants=2, seed=42)
   → treatment/control 50:50 분할

3. 캠페인 실행
   → 처우 그룹: 쿠폰/VIP Care/푸시 발송
   → 통제 그룹: 개입 없음
   → 최소 2주 이상 관찰

4. 통계 검정
   → compute_significance(data, metric="churn_label", alpha=0.05)
   → p-value, 95% CI, 효과 크기 산출

5. 다중 지표 보정
   → MultipleComparisonCorrection.fdr_bh(p_values)
   → 보정된 p-value 기준 유의성 재판단

6. 의사결정
   → 유의미: 캠페인 전체 롤아웃 결정
   → 비유의미: 캠페인 수정 또는 중단
```
