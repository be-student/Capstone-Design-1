# 리텐션 전략 (Retention Strategy)

이탈 위험 고객을 대상으로 세그먼트별 최적화된 리텐션 캠페인을 설계하고 실행하는 전략 문서입니다.
구현 코드: `src/features/segmentation.py`, `src/models/uplift_model.py`, `src/models/budget_optimizer.py`

## 1. 고객 세그멘테이션 체계

### 1.1 운영용 6+ 세그먼트

`run_segment`는 churn probability, Uplift Score, CLV를 결합해 `results/segments_6plus.csv`를 만든다.
RFM 기반 세그먼트는 보조 설명 변수로 유지하지만, 제출용 운영 세그먼트는 아래 6개 이상 체계다.

| 세그먼트 | 기준 | 리텐션 액션 |
|---------|------|------------|
| **high_value_persuadable** | 상위 20% CLV, Uplift 양수, 이탈확률 높음 | VIP Care, 고가치 쿠폰 |
| **mid_value_persuadable** | 40~80% CLV, Uplift 양수, 이탈확률 높음 | 쿠폰 + 개인화 메시지 |
| **low_value_persuadable** | 하위 40% CLV, Uplift 양수, 이탈확률 높음 | 비용 효율 쿠폰/푸시 |
| **high_value_sure_thing** | 상위 20% CLV, 이탈확률 낮음, Uplift 중립 | 최소 접촉, 관계 유지 |
| **mid_value_sure_thing** | 40~80% CLV, 이탈확률 낮음, Uplift 중립 | 자동화 저비용 접점 |
| **low_value_sure_thing** | 하위 40% CLV, 이탈확률 낮음, Uplift 중립 | 자동화 저비용 접점 |
| **high_value_lost_cause** | 상위 20% CLV, 이탈확률 높음, Uplift 중립 | 원인 분석, 고비용 캠페인 보류 |
| **mid_value_lost_cause** | 40~80% CLV, 이탈확률 높음, Uplift 중립 | 원인 분석, 저비용 관찰 |
| **low_value_lost_cause** | 하위 40% CLV, 이탈확률 높음, Uplift 중립 | No Action |
| **sleeping_dog** | Uplift 음수 | 캠페인 제외 |
| **new_customer_onboarding** | 가입 초기 고객 | 온보딩 캠페인 |

### 1.2 Uplift 기반 4-사분면 세그먼트

Uplift Modeling을 통해 추가로 4개의 처우 반응 세그먼트를 도출합니다.

| 세그먼트 | 영문명 | Uplift 점수 | 의미 |
|---------|--------|------------|------|
| **설득 가능** | Persuadables | Uplift > 0.05, 이탈확률 높음 | 처우(캠페인) 시 이탈 방지 효과 큼 |
| **확실한 잔존** | Sure Things | abs(Uplift) ≤ 0.05, 이탈확률 낮음 | 처우 없이도 잔존 가능성 높음 |
| **가망 없음** | Lost Causes | abs(Uplift) ≤ 0.05, 이탈확률 높음 | 처우해도 이탈 방지 효과 미미 |
| **역효과** | Sleeping Dogs | Uplift < 0 | 처우 시 오히려 이탈 촉진 |

## 2. 세그먼트별 리텐션 전략

### 2.1 고가치-Persuadables (VIP충성고객 + 이탈위험고객 + 설득 가능)

**특성:** 과거 높은 구매력, 최근 활동 감소, 캠페인 개입 시 잔존 효과 큼

**전략: VIP Care 프로그램**
- 전담 고객 매니저 연결 (이메일/앱 메시지)
- 개인화된 상품 추천 + 특별 할인 쿠폰 동봉
- VIP 전용 프리미엄 혜택 제공 (무료 배송, 우선 CS)

| 항목 | 내용 |
|------|------|
| **액션 비용** | 70,000원/인 (VIP Care) |
| **예상 Uplift** | 이탈 확률 15~25% p 감소 |
| **목표 CLV 보존** | 1인당 평균 300,000원 이상 |
| **ROI 추정** | (300,000 × 0.20) / 70,000 ≈ 0.86 (86% 수익률) |

### 2.2 고가치-Sure Things (VIP충성고객 + 확실한 잔존)

**특성:** 이탈 가능성 낮고 처우 없이도 잔존. 처우 비용 낭비 가능성 있음

**전략: 최소 개입 + 관계 유지**
- 월 1회 뉴스레터 (개인화 상품 큐레이션)
- 멤버십 포인트 자동 적립 안내
- 과도한 프로모션 메시지 자제 (Sleeping Dog 방지)

| 항목 | 내용 |
|------|------|
| **액션 비용** | 1,000원/인 (푸시 알림) |
| **예상 Uplift** | 이탈 확률 2~5% p 감소 |
| **목표** | 관계 유지 비용 최소화 |
| **ROI 추정** | 비용 최소화가 핵심 |

### 2.3 중간가치-Persuadables (잠재충성고객, 이탈위험고객 + 설득 가능)

**특성:** 중간 구매력, 캠페인에 반응할 가능성 있음

**전략: 쿠폰 + 개인화 메시지**
- 5,000 ~ 10,000원 할인 쿠폰 발급
- 최근 관심 카테고리 기반 상품 추천 푸시
- 재방문 유도 특별 이벤트 참여 안내

| 항목 | 내용 |
|------|------|
| **액션 비용** | 30,000원/인 (쿠폰) |
| **예상 Uplift** | 이탈 확률 8~15% p 감소 |
| **목표 CLV 보존** | 1인당 평균 100,000원 이상 |
| **ROI 추정** | (100,000 × 0.12) / 30,000 ≈ 0.40 (40% 수익률) |

### 2.4 저가치-Persuadables (휴면고객 + 설득 가능)

**특성:** 낮은 구매력이지만 적절한 개입으로 재활성화 가능

**전략: 재활성화 푸시 + 소액 인센티브**
- 앱 푸시 알림: "오랫동안 보지 못했어요! 돌아오세요"
- 소액 포인트 지급 (1,000~3,000원)
- 계절/이벤트 연계 쇼핑 동기 부여

| 항목 | 내용 |
|------|------|
| **액션 비용** | 1,000원/인 (푸시) + 소액 포인트 |
| **예상 Uplift** | 이탈 확률 5~10% p 감소 |
| **목표** | 저비용 재활성화 |

### 2.5 Lost Causes (가망 없음 세그먼트)

**특성:** 처우해도 이탈 방지 효과 미미

**전략: 비용 투자 최소화**
- 캠페인 제외 (예산 절약)
- 브랜드 인지도 유지 목적의 최소 접점만 유지
- 향후 재진입 가능성을 위한 데이터 보존

| 항목 | 내용 |
|------|------|
| **액션 비용** | 0원 (캠페인 제외) |
| **전략** | 이탈 허용 후 재획득 비용 검토 |

### 2.6 Sleeping Dogs (역효과 세그먼트)

**특성:** 캠페인 노출 시 오히려 이탈 확률 증가

**전략: 의도적 접촉 회피**
- 모든 마케팅 커뮤니케이션에서 제외
- 자연 이탈 관찰 후 재획득 캠페인으로 전환 검토
- 이 세그먼트에 대한 접촉은 반드시 사전 검증 필요

| 항목 | 내용 |
|------|------|
| **액션 비용** | 0원 |
| **전략** | 접촉 금지. 역효과 방지가 최우선 |

## 4. Uplift Modeling 적용 과정

### 4.1 T-Learner (Two-Model Approach)

T-Learner는 처우 그룹과 통제 그룹에 **별도 모델**을 학습하여 CATE(Conditional Average Treatment Effect)를 추정합니다.

```
1단계: 처우 그룹(T=1)으로 μ₁(x) 학습
       P(Y=1 | X=x, T=1) → 처우받은 경우의 이탈 확률

2단계: 통제 그룹(T=0)으로 μ₀(x) 학습
       P(Y=1 | X=x, T=0) → 처우받지 않은 경우의 이탈 확률

3단계: Uplift = μ₀(x) - μ₁(x)
       처우로 인한 이탈 확률 감소량 (양수 = 캠페인 효과 있음)
```

기반 모델: `GradientBoostingClassifier(n_estimators=100, max_depth=4, learning_rate=0.1, min_samples_leaf=10)`

### 4.2 S-Learner (Single-Model Approach)

S-Learner는 처우 지시자를 **하나의 피처로 포함**하여 단일 모델을 학습합니다.

```
1단계: [X, T]를 입력으로 μ(x, t) 학습

2단계: Uplift = μ(x, T=0) - μ(x, T=1)
       처우 지시자만 변경하여 차이 계산
```

### 4.3 T-Learner vs S-Learner 비교

| 항목 | T-Learner | S-Learner |
|------|----------|----------|
| **모델 수** | 2개 (처우/통제 별도) | 1개 |
| **장점** | 처우/통제 그룹 특성 독립 학습 가능 | 데이터 효율적, 그룹 간 공통 패턴 활용 |
| **단점** | 소규모 그룹에서 분산 증가 | 처우 효과를 과소 추정할 수 있음 |
| **권장 상황** | 처우/통제 그룹이 충분히 클 때 | 처우 그룹이 작을 때 |
| **기본 설정** | `learner="t_learner"` (기본값) | `learner="s_learner"` |

## 5. 예산 최적화 수식

### 5.1 목적 함수

이탈 방지로 보존되는 기대 CLV를 최대화합니다:

$$\text{Maximize} \sum_{i} \text{Uplift}_i \times \text{CLV}_i \times \text{Action}_i$$

$$\text{Subject to} \sum_{i} \text{Cost}_i \times \text{Action}_i \leq \text{Budget}$$

$$0 \leq \text{Action}_i \leq 1 \quad \forall i$$

### 5.2 LP 정식화

`BudgetOptimizer`는 `scipy.optimize.linprog`로 선형 계획법(LP) 풀이:

```
결정 변수: x_i ∈ [0, cost_per_action_i]  (고객 i에 투자할 예산)

목적 함수: maximize  Σ_i (priority_i / cost_i) × x_i
           = minimize Σ_i -(priority_i / cost_i) × x_i

여기서 priority_i = max(uplift_i, 0) × clv_i × churn_prob_i

제약 조건:
  Σ_i x_i ≤ total_budget    (총 예산 제약)
  0 ≤ x_i ≤ cost_per_action_i  (개인별 상한)
```

### 5.3 우선순위 점수 해석

- **Uplift**: 캠페인의 실제 효과 (처우-통제 차이)
- **CLV**: 고객 생애 가치 (이탈 방지 시 보존 가치)
- **Churn Probability**: 이탈 위험도 (높을수록 개입 긴급도 높음)

세 요소의 곱이 높은 고객 = **캠페인 투자 대비 기대 수익이 가장 높은 고객**

### 5.4 예산 시나리오 예시

총 예산 5,000만원 가정:

| 세그먼트 | 대상 고객 수 | 단가 | 소요 예산 | 기대 Uplift |
|---------|-----------|------|---------|------------|
| VIP Care | 100명 | 70,000원 | 7,000,000원 | 이탈 20% 감소 |
| 쿠폰 발급 | 500명 | 30,000원 | 15,000,000원 | 이탈 12% 감소 |
| 푸시 알림 | 28,000명 | 1,000원 | 28,000,000원 | 이탈 5% 감소 |
| **합계** | **28,600명** | - | **50,000,000원** | - |

LP 최적화는 위 예산을 고정하지 않고 priority_i/cost_i 비율에 따라 동적으로 배분합니다.
