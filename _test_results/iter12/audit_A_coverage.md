## Recommendations (audit-grade actions)

1. **Block the "iter9 P0s are fixed" claim until `src/dashboard/calculations.py` reaches >=95 %.** The five fix helpers added for the float-spill / ROI / customer-count contradictions are pure functions; the tests are trivial. Today they exist but are bypassable without any test failing.
2. **Add page-render coverage for `src/dashboard/app.py`.** Even thin smoke tests calling each `_render_pageNN(...)` with a stub data_loader would lift this from 41 % to ~75 %. Focus the *first* tests on the confusion-matrix reconciliation block (lines 439–488) — that block alone is the iter9 P02 #1 patch and is currently asserted only by inspection.
3. **Reconcile `tests/test_model_monitoring_view.py` with `src/dashboard/monitoring_view.py`.** The 7 failures are not a regression in monitoring behavior — they are stale tests pointing at a never-merged "Survival Curves" redesign. Either restore `_render_survival_section` + the header rename, or delete those test classes. As-is, "7 failed tests" gives a false impression of regression.
4. **Promote `src/main.py` integration coverage past 70 %.** A single end-to-end pipeline test that runs the orchestrator against tiny config and asserts manifest contents would cover ~30 % of the remaining gap in one shot.
5. **Fill `src/dashboard/data_loader.py` MLflow / drift / segment-name loaders.** Adding tests for the loader branches that feed Page 11, Page 14, Page 15 will directly catch the cross-page contradictions (iter9 P0 #3, #4, #6) at the level where they originate.
6. **Delete or implement `src/models/survival_model.py`** — 2-line stub at 0 %. Either re-export `survival_analysis` or remove from the tree.
7. **Install `pytest-timeout`** so future audits can run with `--timeout=900` as specified. Without it, a runaway test can hang the whole suite (didn't happen here — full run was 278 s — but the audit spec assumed it would be installed).
